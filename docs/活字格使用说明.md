# TalentFlow ATS 活字格工程使用说明

## 文件说明

| 文件 | 说明 |
|------|------|
| `TalentFlow-ATS.fgcc` | 活字格工程文件（可直接用设计器打开） |
| `Document.fgcc` | 原始模板（第 8 组参考工程） |
| `docs/build_talentflow_fgcc.py` | 重新生成 fgcc 的脚本 |

`.fgcc` 本质是 **ZIP 压缩包**，内含页面 JSON、数据表定义、SQLite 数据库（`ForguncyDB.sqlite3`）、插件与 ServerCommands 等。

## 打开方式

1. 安装 **活字格设计器**（Forguncy Designer，建议 v11+）
2. 设计器 → **打开** → 选择项目根目录下的 `TalentFlow-ATS.fgcc`
3. 若提示「版本不一致」，选择 **忽略** 或 **升级文档** 后继续
4. 点击 **本地运行** 预览（默认网站端口 **23510**）

## 工程结构（与 Document.fgcc 同格式）

```
TalentFlow-ATS.fgcc (ZIP)
├── ForguncyDB.sqlite3          # 内置数据库
├── ApplicationSettings.json    # 应用配置（起始页 submit）
├── Pages/
│   ├── submit.json             # 简历提交 + AI 初筛
│   ├── result.json             # 评审结果
│   └── history.json            # 历史记录列表
├── Tables/
│   ├── review_record_.json     # AI 评审记录表
│   ├── position_.json          # 岗位表（新增）
│   └── application_.json       # 投递表（新增）
├── ServerCommands/
│   ├── agent.json              # AI 对话评审
│   ├── workflow.json           # 工作流
│   └── saveReviewRecordToDatabase.json
└── Plugin/                     # AI 对话等插件
```

## 与本项目 Spring Boot 版的对应关系

| 活字格表/页面 | Spring Boot 模块 | 说明 |
|---------------|------------------|------|
| `position_` | `Position` 实体 | 岗位 title/status/department |
| `application_` | `Application` 实体 | 投递 stage/match_score/channel |
| `review_record_` | `AiMatchingService` | AI 简历评审结果入库 |
| submit/result/history | C 端智能投递 | 低代码演示 AI 初筛流程 |
| 内置页面（登录/岗位列表等） | 各角色 Vue 页面 | 模板自带，可在设计器中扩展 |

## 重新生成 fgcc

修改脚本或模板后，在项目根目录执行：

```bash
python docs/build_talentflow_fgcc.py
```

将在根目录生成新的 `TalentFlow-ATS.fgcc`。

## 注意事项

1. **AI 插件**：模板依赖「AI 对话助手」插件，需在设计器中能正常加载；AI 功能需在活字格中配置模型/MCP（与 Spring Boot 版的规则引擎不同）。
2. **完整三端功能**（C/B/M 漏斗、管理驾驶舱）在 **Vue + Spring Boot** 版中实现，用 `start-all.bat` 启动 http://localhost:5180。
3. 在设计器中新增表/页面后，请 **保存为 .fgcc** 覆盖或另存工程文件。
